# Contributing to AI Engineering Playbook

Thank you for your interest in contributing to this playbook! This is primarily a personal engineering journal, but contributions, suggestions, and discussions are welcome.

## How to Contribute

### 1. Reporting Issues

If you find errors, broken links, or have suggestions:

1. Check existing issues first
2. Create a new issue with:
   - Clear description of the problem
   - Steps to reproduce (if applicable)
   - Suggested solution (if you have one)

### 2. Suggesting Content

Have an idea for new content or improvements?

1. Open an issue with the `enhancement` label
2. Describe what you'd like to see added
3. Explain why it would be valuable

### 3. Pull Requests

#### Small Fixes
For typos, broken links, or minor corrections:
- Fork the repository
- Make your changes
- Submit a PR with a clear description

#### New Content
For substantial additions:
1. Open an issue first to discuss
2. Wait for feedback/approval
3. Fork and create a feature branch
4. Follow the content guidelines below
5. Submit a PR

## Content Guidelines

### Structure

Each major topic should follow this structure:

```
topic-name/
├── README.md          # Main content
├── examples/          # Code samples
│   ├── example1.cs
│   └── example2.cs
└── resources.md       # Additional references
```

### Markdown Style

- Use clear, descriptive headings
- Include code examples with syntax highlighting
- Add tables for comparisons
- Use bullet points for lists
- Include links to official documentation

### Code Examples

- Provide complete, runnable examples when possible
- Include comments for complex logic
- Follow C# coding conventions
- Test code before submitting
- Show both basic and advanced usage

### Documentation Standards

Each README should include:

1. **Objective**: What the topic covers
2. **Core Concepts**: Key ideas and patterns
3. **Implementation Examples**: Practical code
4. **Results & Learnings**: Findings and insights
5. **Best Practices**: Recommendations
6. **Resources**: Further reading

## Example Template

```markdown
# Topic Name

## Objective
What this covers and why it matters.

## Core Concepts
Key ideas with examples.

## Implementation
Working code examples.

## Results & Learnings
Findings from practical application.

## Best Practices
Recommended approaches.

## Resources
- [Link 1](url)
- [Link 2](url)
```

## Code of Conduct

### Expected Behavior

- Be respectful and constructive
- Focus on the content and ideas
- Provide helpful feedback
- Give credit where it's due

### Unacceptable Behavior

- Harassment or discrimination
- Spam or off-topic content
- Plagiarism
- Malicious code or suggestions

## Getting Help

- Open an issue for questions
- Use discussions for open-ended topics
- Tag issues appropriately
